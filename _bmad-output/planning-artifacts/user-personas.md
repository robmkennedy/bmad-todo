## Overview

This document defines the primary user personas for BMad Todo. These personas are based on the PRD's target user definition and represent the key user archetypes who will benefit from a zero-friction, immediately usable task management experience.

Since BMad Todo v1.0 focuses on **individual users** with **no authentication**, our personas reflect people who need simple, reliable personal task management without complexity.

---

## Persona 1: Maya Chen — The Busy Professional

### Demographics

| Attribute | Value |
|-----------|-------|
| **Age** | 34 |
| **Occupation** | Marketing Manager at a mid-size tech company |
| **Location** | Austin, TX |
| **Tech Comfort** | High — uses 10+ apps daily |
| **Devices** | MacBook Pro, iPhone 15, iPad |

### Photo Placeholder
🎯 *Professional woman, confident, multitasking*

### Quote
> "I don't have time to learn another tool. I just need to write something down before I forget it."

### Background

Maya juggles multiple campaigns, client meetings, and cross-functional projects. Her days are fragmented — context-switching between Slack, email, Figma reviews, and video calls. She's tried sophisticated project management tools but finds them overkill for personal task tracking. She often resorts to sticky notes or random text files, which inevitably get lost.

### Goals

| Goal | Priority |
|------|----------|
| Capture tasks instantly without friction | 🔴 Critical |
| See what needs to be done at a glance | 🔴 Critical |
| Clear completed items to feel progress | 🟡 Important |
| Access tasks from any device | 🟡 Important |

### Pain Points

| Pain Point | Severity |
|------------|----------|
| Tools that require account creation before use | 🔴 High |
| Complex interfaces with features she'll never use | 🔴 High |
| Losing track of tasks scattered across apps | 🟡 Medium |
| Forgetting tasks captured in the wrong place | 🟡 Medium |

### Behaviors

- **Morning ritual**: Reviews tasks with coffee before diving into email
- **Capture pattern**: Jots tasks during meetings, on the go, between calls
- **Completion pattern**: Checks off tasks immediately for dopamine hit
- **Review pattern**: Quick scan at end of day to prep for tomorrow

### Technology Usage

| Tool | Purpose | Satisfaction |
|------|---------|--------------|
| Apple Reminders | Personal tasks | 😐 "Too basic, syncing issues" |
| Asana | Team projects | 😤 "Overkill for my personal stuff" |
| Sticky Notes | Quick capture | 😫 "Always lose them" |
| Notes app | Random lists | 😕 "Becomes a mess" |

### Success Metrics

Maya would consider BMad Todo successful if:
- ✅ She can add a task in under 5 seconds
- ✅ She stops using sticky notes
- ✅ She can access her list on phone and laptop
- ✅ She never loses a task again

### Design Implications

| Insight | Design Response |
|---------|-----------------|
| Zero patience for onboarding | No signup, no tutorial — just a text field |
| Needs instant capture | Input field always visible, Enter to submit |
| Visual progress matters | Clear distinction between done/not done |
| Mobile is critical | Fully responsive, touch-friendly |

## Persona 2: David Park — The Focused Minimalist

### Demographics

| Attribute | Value |
|-----------|-------|
| **Age** | 28 |
| **Occupation** | Freelance Graphic Designer |
| **Location** | Portland, OR |
| **Tech Comfort** | Medium-High — prefers simple tools |
| **Devices** | iMac, Android phone |

### Photo Placeholder
🧘 *Creative professional, calm, intentional*

### Quote
> "Every app wants to be everything. I just want a list that works."

### Background

David left agency life for freelance work, valuing autonomy and focus. He's deliberately minimalist — in his apartment, his wardrobe, and his tools. He's tried Notion, Todoist, and Things, but found himself spending more time organizing his system than actually doing work. He believes constraints breed creativity and seeks tools that do one thing well.

### Goals

| Goal | Priority |
|------|----------|
| Maintain a single, uncluttered task list | 🔴 Critical |
| Reduce digital noise and distractions | 🔴 Critical |
| Feel calm when looking at his tasks | 🟡 Important |
| Finish what he starts before adding more | 🟢 Nice to have |

### Pain Points

| Pain Point | Severity |
|------------|----------|
| Feature bloat in productivity apps | 🔴 High |
| Notifications and reminders creating anxiety | 🔴 High |
| Tools that encourage over-organization | 🟡 Medium |
| Aesthetically unpleasant interfaces | 🟡 Medium |

### Behaviors

- **Morning ritual**: Reviews list, picks 3 things to focus on
- **Capture pattern**: Deliberate — only adds tasks he'll actually do
- **Completion pattern**: Savors checking things off, reviews at end of day
- **Review pattern**: Weekly cleanup to remove stale items

### Technology Usage

| Tool | Purpose | Satisfaction |
|------|---------|--------------|
| Paper notebook | Daily tasks | 😊 "Love it, but not searchable" |
| Things 3 | Tried it | 😐 "Beautiful but too many features" |
| Apple Notes | Quick capture | 😕 "Not a real task list" |

### Success Metrics

David would consider BMad Todo successful if:
- ✅ The interface is clean and calming
- ✅ There are no unnecessary features
- ✅ He feels in control, not overwhelmed
- ✅ It loads instantly with no splash screens

### Design Implications

| Insight | Design Response |
|---------|-----------------|
| Values aesthetics + simplicity | Clean typography, ample whitespace |
| Dislikes feature creep | Resist adding "nice to have" features |
| Wants calm, not urgency | No aggressive colors, no notifications |
| Appreciates craft | Polish the details — transitions, spacing |

## Design Principles Derived from Personas

Based on these personas, BMad Todo should embody:

### 1. ⚡ Instant Usability
> No signup, no onboarding, no learning curve. Open → Use.

All three personas value immediacy. Maya has no patience, David wants simplicity, Jordan wants speed.

### 2. 🎯 Focused Functionality
> Do one thing exceptionally well. Resist feature creep.

David will abandon us if we add too much. Maya will ignore features she doesn't need. Jordan might like more features but doesn't need them in v1.

### 3. 📱 Device Agnostic
> Work beautifully on desktop and mobile, with no compromises.

Maya switches devices constantly. Jordan captures on mobile. David works primarily on desktop. All must feel native.

### 4. 👁️ Visual Clarity
> The task list should be scannable in 2 seconds.

All personas do "quick scans." Visual hierarchy must be immediate — what's done, what's not, what's most recent.

### 5. 🧘 Calm Confidence
> Users should feel in control, not anxious.

David explicitly wants calm. Maya wants reliability. Jordan wants to feel accomplished. No aggressive urgency.
